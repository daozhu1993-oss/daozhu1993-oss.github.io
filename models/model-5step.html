一句话总览： 五步拆解法是一套「目标 → 问题 → 诊断 → 方案 → 执行」的实战流程——不靠热血，靠流程，把任何吓人的复杂目标，拆成每天能动手的具体模块。它适用于你想达成的任何一件大事。
Origin
岛主 6 个月拆解 100 个模型的实战复盘根植于目标管理（Goal Setting）与执行方法论。
Field
目标管理 / 内容创作 / 个人成长任何"看起来吓人"的复杂目标都能套用。
Core Definition
五步闭环 G·P·D·S·EGoal 目标 / Problem 问题 / Diagnose 诊断 / Solution 方案 / Execute 执行。
Thinking Layer
执行落地层在"想清楚"之后，专门解决"怎么干成"。

喊口号 vs 五步拆解

Wish vs System
✕

靠热血的Flag

Vibes-Only Goal
  • 立下"我要变厉害"的模糊愿望，等奇迹发生
  • 问题一冒头就焦虑、逃避，假装看不见
  • 停在表面："最近没时间"成了万能借口
  • 方案是"我要努力"这种空话，三天打鱼
例：年初立 Flag"今年要多分享思维模型"，半年后还是 0 篇——因为从没把它拆开。
✓

五步拆解流程

Goal · Problem · Diagnose · Solution · Execute
  • 目标可衡量："6 个月 100 篇"，不是"变厉害"
  • 主动把拦路虎摊在桌上，不逃避
  • 深挖病根：流程缺陷、预期偏差、资源散落
  • 方案具体到动作，用固定时段+复盘执行
例：每周日批量选题、固定早 9–11 点创作、周五复盘——把"100 篇"变成每天 2 小时的具体模块。

五个关键动作

The Five Moves
01

目标 Goal

不是"我想"，而是"我必须达成"。要可衡量（100 篇 / 6 个月）、有意义、不自我设限——先接受挑战，再找路径。

例："6 个月发布 100 篇模型拆解，形成系统性合集"，而不是"我要多分享"。
02

问题 Problem

目标刚立下，拦路虎就扑面而来。把它们全摊在桌上：时间够吗、质量如何保、流程能否标准化、反馈对不对准用户。

03

诊断 Diagnose

别停在表面，深挖为什么。病根常是三类：流程缺陷（没建生产线）、预期偏差（追求完美）、资源未整合（素材散落）。

例："没时间"的真相，是每篇从零循环、边际成本极高。
04

方案 Solution

设计你的"生产线"：批量选题、模板化生产、流程切片（找资料 / 制图 / 写文案分开批量做），再加知识库随时调取。

05

执行 Execute

规划再好不执行也白搭。固定时段雷打不动、周度复盘卡点、公开承诺借外部监督倒逼自己。

例：每天早 9–11 点创作，周五查进度与数据，公开目标。

套到你的目标上

Apply It to Your Flag

写下"必须达成"

把愿望改成可衡量的目标：数量、时间、频率，一句能验收的话。

列出所有拦路虎

不逃避，把时间、质量、流程、反馈的问题全摊开，写满一页。

追问三次"为什么"

对每个问题挖病根，找到是流程、心态还是资源的问题。

设计具体生产线

把大任务切成可复用的模块，定好模板与批量处理的节奏。

固定时段+复盘

雷打不动的创作时段，每周复盘卡顿，再公开承诺加一道锁。

模型金句

In a Nutshell

目标不是用来设立的，是用来拆解的。

五步流程的核心信念

大多数失败，卡在"诊断"环节——我们总在解决表面问题，却从不深挖机器为什么失灵。

岛主复盘

"我要努力"不是方案，"我每周日晚上花 1 小时批量选题"才是。

方案要具体
