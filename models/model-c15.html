报酬矩阵

Payoff Matrix

别拿"我觉得行"拍板，用期望收益算一算

AND 株式会社《商业框架图鉴》 ·  决策分析工具
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 报酬矩阵——把每个方案在不同结果下的"赚多少/亏多少"列成一张表，再按概率加权平均，比出最优解。它源自日本 AND 株式会社的《商业框架图鉴》，是给"拍脑门决策"的一剂退烧药，专治"我觉得稳了"这种病。
Origin
AND 株式会社《商业框架图鉴》日本 AND 株式会社整理的商业框架工具书，系统收录实用决策模型。
Field
决策 / 商业分析凡是要在多个方案里选一个的场合都用得上。
Core Definition
期望收益 = Σ(收益 × 概率)把"该选哪个"交给加权平均的算术，而不是直觉。
Thinking Layer
量化决策层在直觉之上，再叠加一道概率与期望的冷计算。

拍脑门 vs 报酬矩阵

Bad vs Good
✕

拍脑门决策

Gut Feeling Only
  • 凭"直觉"就拍板上项目
  • 总在赌运气，而不是算清期望值
  • 最好情况一想就上头，落地却亏到底
  • 不量化，决策像蒙眼扔飞镖
例：看"最好赚 100 万"就 All in，结果那剧本只有 10% 概率，扎不中怪手臭。
✓

报酬矩阵

Expected Payoff
  • 列损益、算期望、比均值
  • 看期望，别被最好情况晃花眼
  • 再用"难度 × 效果"二次筛选
  • 好方案是平均最稳的，不是最刺激的
例：算完期望，稳赚 20 万的 B 方案完胜，事后证明躲过一劫，A 那家半年就黄了。

四个关键要点

The Four Moves
01

设定收益

先给每个方案在不同自然状态下标上损益值。不量化，决策就是拍脑门，像蒙眼扔飞镖，扎不扎中全凭命。

02

加权期望

按客观概率算出每个方案的期望收益值。看期望，别看最好情况——最好的剧本往往概率最小，盯着它的人最后都成了分母。

03

择优比较

期望值最高的，就是该选的。好方案是"平均下来"最稳的那个，不是最刺激的，刺激的多半是陷阱，糖衣里裹着亏空。

04

难度 × 效果

效果好 + 可行性高优先做；又难又没效的坑，直接跳过，别恋战，省下的力气去攻能啃下的山头。

应用流程

From Guess to Expected Value

列出结果

给每个候选方案列出"最好 / 最差 / 一般"三种结果和各自概率，先把账摊开。

算出期望

加权平均算期望收益，别被最好情况诱惑——均值才是该看的那个数。

择优比较

选期望值最高的方案，而不是最刺激的。理性人选均值，不押小概率的烟花。

难度 × 效果再筛

用"效果 × 可行性"排优先级，又难又没效的坑直接跳过，难的往后排。

马上能用的事

Put It to Work
✓

重大决定先拉一张矩阵

别凭直觉，把每个方案的收益和概率老老实实列出来。

✓

算出期望再拍板

看均值，别被"最好赚 100 万"晃花眼，把家底押在小概率上最危险。

✓

用"效果 × 可行性"再筛一遍

又难又没效的坑直接跳过，省下力气攻能啃下的山头。

✓

给概率找客观依据

别拍脑袋给概率，查数据、看历史，概率越实期望越准。

✓

把"最好情况"单独标红

提醒自己它概率最小，别让最好剧本替你做决定。
