四要素模型

Four-Factor Model
三谷宏治  ·  Koji Mitani  ·  早稻田大学 /《商业模式全史》

你以为在谈客户，其实在谈一张价值网——谁付钱、谁用产品，常不是同一张脸。

一句话总览： 四要素模型是早稻田大学三谷宏治在《商业模式全史》里提的朴素判断——商业模式不是"客户的事"，而是"总价值创造的事"。你以为在谈客户，其实在谈一张让所有相关方都分到价值的网。
Origin
三谷宏治Koji Mitani，早稻田大学 /《商业模式全史》。把商业模式拆成"价值网"而非"单点客户"。
Field
商业模式 / 价值网商业模式设计、盈利模式梳理、平台与利益相关方管理。
Core Definition
织一张价值网顾客/价值提供、盈利/资源、利益相关者，全分到价值才立住。
Thinking Layer
商业模式 / 系统层在"做什么"之外，解决"怎么让各方都不想走"。

只盯客户 vs 织一张价值网

Bad vs Good
✕

只盯客户

Chase the Customer
  • 总盯着"怎么拿下这个客户"
  • 越想讨好用户，利润越薄
  • 忘了谁在背后给你搭台
  • 把商业模式当成"搞定客户"一件事
例：只列客户清单，忽略广告商替用户买单的链路。
✓

织一张价值网

Weave the Value Web
  • 画全利益相关者清单，不只列客户
  • 问每个参与者今天得到什么价值
  • 找出"谁替谁买单"的隐性链路
  • 让所有相关方都分到价值，网才立住
例：吉列刀架便宜卖、刀片持续赚，把用户和收入流拆成两拨人。

四个关键要素

The Four Factors
01

顾客 + 价值提供

你给客户什么，这是钩子。但光有钩子不够，客户愿意掏钱只是价值网的第一环。

02

盈利方式 + 战略资源

怎么收钱、靠什么收钱，是商业模式的心脏。没有现金流的价值主张都是空中楼阁。

03

利益相关者

创造者、经销商、对手，甚至广告商——让所有相关方都分到价值，这张网才立得住。

04

隐性链路

一张网里，谁付钱、谁用产品，常常不是同一张脸。吉列刀片-刀架、免费-收费，都是把两拨人拆开。

应用流程

From Customer to Web

画利益相关者清单

别只列客户，把创造者、经销商、对手、广告商都写上。

问每方得到什么价值

今天每个参与者从这张网里拿走了什么？

找"谁替谁买单"的链路

用户和收入流常不是同一张脸，隐性链路才是利润所在。

设计让各方都不想走的网

商业不是零和抢客户，而是搭一张留人的网。
