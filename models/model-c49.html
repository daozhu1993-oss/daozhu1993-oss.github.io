上瘾模型

Hook Model
尼尔·埃亚尔  ·  《上瘾》  ·  行为设计
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 上瘾模型是尼尔·埃亚尔在《上瘾》里拆出的四步循环——"触发—行动—多变的酬赏—投入"。它专讲产品怎么让你心甘情愿回来：让人上瘾的不是产品多好，是他自己投进去的多少。
Origin
尼尔·埃亚尔《上瘾》(Hooked) 里拆出的行为设计四步循环。
Field
产品设计 / 用户运营 / 行为设计凡是要"让用户回来"的场合都用得上。
Core Definition
四步循环触发 Trigger·行动 Action·多变的酬赏 Reward·投入 Investment。
Thinking Layer

一次性满足 vs 上瘾循环

Bad vs Good
✕

一次性满足思维

Use Once, Gone
  • 只做一次性满足，没有"下次还来"的钩子
  • 用户行动门槛高，一步变两步
  • 奖励太确定，用几次就腻
  • 不攒"投入"，用户说走就走
例：确定性奖励很快腻；用户没投入，卸载毫无心理负担。
✓

上瘾循环思维

The Hook Loop
  • 触发分内外，熬成内部习惯最黏
  • 行动门槛砍到最低（B=MAT）
  • 酬赏要"多变"，像拆盲盒
  • 让用户投入时间/数据/关注者
例：小红书你发一条不知谁赞多少——多变酬赏；攒百粉写五十篇，已"投入"，轻易不注销。

上瘾四步拆解

The Four Steps
T

触发 Trigger

分内外。内部是情绪缺口（无聊、焦虑），外部是广告推送。真正黏人的，是把外部熬成内部习惯。

A

行动 Action

行为=动机×能力×触发（B=MAT）。用户懒是最大敌人，能一步点的别让他点两步。

R

多变的酬赏 Reward

猎物（新信息）/社交（被赞）/自我（成就感）。关键在"多变"——越随机越上头。

公式：越 predictable 越无聊，越随机越上头。
I

投入 Investment

时间、数据、关注者。越塞越不舍得扔，投入和重视成正比——这是人性。

应用流程

Build Your Hook

画自己的上瘾循环

砍到最低行动门槛

设计"下次还来"的钩子

多变酬赏，而非一次性满足。

攒用户的投入

让时间/数据/关注者沉淀下来，越塞越不舍得走。
