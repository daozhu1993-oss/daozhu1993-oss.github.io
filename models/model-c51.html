领导梯队模型

Leadership Pipeline
升职不是多做一点，是换一种活法
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 人从干活到带人、带团队、带生意，要跨过六道坎——每上一阶，你"该操心的事"彻底变一遍。领导梯队模型讲的就是这种"换脑子"的规律：不是能力不够，是没换活法。
Origin
岛主解读 · 领导力发展经典框架归纳自组织发展与领导力晋升的通用规律，源流见《领导梯队》体系。
Field
领导力 / 组织发展 / 管理晋升凡是从"自己干"迈向"带人带团队带生意"的台阶。
Core Definition
六阶领导力跃迁每上一阶，职责重心彻底切换，不是多做一点。
Thinking Layer
角色认知层 / 组织层级解决"我在这一阶，到底该操心什么"。

业绩好≠能当官 vs 换一种活法

Bad vs Good
✕

把"业绩好"等于"能当官"

Same Game, Higher Title
  • 技术大牛提拔成主管，半年就垮
  • 越往上走，老本事越不灵，还硬用
  • 自己上手抢活，团队永远长不大
  • 用管组员的方式管主管
例：销售冠军升区域经理，第一月天天自己跑客户，团队反而散了。
✓

每升一阶，换一种活法

Switch the Game
  • 从"我自己干完"变"让别人干完"
  • 从带兵到带"带兵的人"，看梯队
  • 从"把事做对"切到"做对的事"
  • 专才到通才，为 P&L 全盘负责
例：KPI 早该从"我卖多少"变成"他们卖多少"。

六道坎，每阶换的脑子

The Six Passages
01

管理自我 → 管理他人

从"我自己干完"变成"让别人干完"。像 solo 弹琴到当指挥。最大坑：自己上手抢活，团队永远长不大。

02

管理他人 → 管理经理

你不再带兵，改带"带兵的人"，看梯队健不健康。别用管组员的方式管主管。

03

管理经理 → 管理职能

开始对一块业务结果负责，会抢资源、定标准。从"把事做对"切到"做对的事"。

04

职能 → 事业部总经理

第一次为 P&L 全盘负责，钱人产品一把抓。专才到此易翻车——因为要当通才。

05

事业部 → 集团高管

平衡长短利益，培育软实力。不是自己赢，是让所有事业部都赢。

06

集团 → CEO

定方向、布文化、对外负责。像船长，风暴里掌舵。

应用流程

From Player to Captain

每升一级先停老动作

先问"我该停止做什么老动作"，别把上一阶的本事直接搬上来。

别拿旧尺子量新人

别用上一阶的标准要求下一阶的人，否则双方都别扭。

帮下属换脑子

帮下属看清他下一阶要换的脑子，梯队才接得上。

调自己的 KPI

把 KPI 从"我干多少"调成"他们干多少"，你才真上了台阶。

模型金句

In a Nutshell

领导梯队的本质

最大的坑，是自己上手抢活，团队永远长不大。

第一道坎的陷阱

马上能用的事

Put It to Work
✓

每升一级，先列"该停的老动作"

不甩掉旧活法，新位子坐不稳。

✓

别用旧标准卡下一阶的人

用上一阶的尺子量新人，只会互相折磨。

✓

帮下属看清下一阶的脑子

梯队接得上，你才能往更高处走。

✓

把 KPI 调成"他们卖多少"

还盯"我卖多少"，说明你没真升上去。

✓

带主管时看梯队健康

别自己下场，看他们带的人长没长起来。

✓

到了通才阶，补不熟的板块

专才升总经理想不翻车，主动补钱人产品的短板。

什么时候该掏出它

When to Use
新晋管理者 技术转管理 晋升过渡 组织发展 团队梯队 空降高管 任何"带人带团队带生意"的台阶
