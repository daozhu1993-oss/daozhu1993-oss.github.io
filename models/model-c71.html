六个盒子

Six Boxes
团队病了却说不清哪疼？用六盒体检，病灶自己浮出来。
马文·韦斯伯德  ·  Marvin Weisbord  ·  1976 年组织诊断框架
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 六个盒子把组织当成一个整体来「体检」——目的、结构、关系、奖励、支持、领导，六个盒子跑一遍，哪一盒漏风，病灶自己就浮出来。它不替你开药，但能让你说清「病到底在哪」。
Origin
马文·韦斯伯德Marvin Weisbord，1976 年提出，组织发展（OD）经典诊断工具。
Field
组织诊断 / 团队管理团队一变就乱、说不清问题出在哪时，用它整体看。
Core Definition
六个维度体检目的·结构·关系·奖励·支持·领导，逐一过一遍。
Thinking Layer
系统层 / 整体诊断在「头痛医头」之前，先整体看一遍组织。

头痛医头 vs 六盒体检

Bad vs Good
✕

头痛医头式管理

Whack-a-Mole
  • 哪里出问题就改哪里，从没整体看过组织
  • 业务一变，组织乱成一锅粥，却说不清病在哪
  • 猛加 KPI、乱调结构，治标不治本
  • 单点动手，其他盒子跟着感冒
例：业绩下滑，老板猛加 KPI（只动奖励盒），人更慌，问题却更偏。
✓

六盒整体体检

Whole-System Checkup
  • 把组织当整体，六个盒子逐一过，不漏项
  • 从「目的盒」问起，先校准方向再调其他
  • 看清盒与盒的牵连：一盒漏风，五盒跟着感冒
  • 领导盒当巡检员，定期查五个盒子漏没漏
例：业绩下滑一跑六盒，才发现目的盒早模糊了——大家根本不知在为谁创造价值。

六个盒子拆解

The Six Boxes
01

目的盒 Purpose

我们为谁创造什么价值？方向不清晰，后面五个盒子全在瞎忙。

02

结构 / 关系盒

怎么分工、怎么处理冲突。分工是骨架，冲突处理是韧带，断一个就瘸。

03

奖励盒 Reward

靠什么激发动力。钱给错地方，好人也会变懒。

04

支持盒 Support

有没有协调手段、配套工具。没支持，再好的结构也转不动。

05

领导盒 Leader

最核心。老板在干啥？职责就是盯紧另外五个别漏风。

06

六盒联动

组织病从不是单点故障，一盒漏风，五盒跟着感冒。

落地五步

Run the Checkup

变化即体检

业务或架构一变动，立刻六盒跑一遍做体检，别等问题炸了才看。

从目的盒问起

先校准方向：我们到底为谁创造什么价值？方向不清，调啥都偏。

查结构关系盒

分工清不清、冲突怎么处理的，骨架韧带断没断。

查奖励支持盒

动力给对没、协调手段够不够，钱有没有给错地方。

领导盒巡检

把领导盒当巡检员，定期查五个盒子漏没漏风。

Put It to Work
✓

团队一乱，先别急着改

六盒跑一遍再说，说清病到底在哪，再动手不迟。

✓

写一句「目的盒」

我们为谁创造什么价值？方向校准了，其他才好调。

✓

业绩掉了别只加 KPI

先查目的是不是模糊了——奖励加错地方，越加越偏。

✓

领导盒当巡检员

每月查五个盒子漏没漏风，别等出了事才巡。

✓

调结构前想清关系盒

冲突处理谁来接？骨架韧带断了，分工再漂亮也瘸。

✓

奖励调整前问一句

这钱会不会给错地方？好人变懒，往往是从这儿开始的。
