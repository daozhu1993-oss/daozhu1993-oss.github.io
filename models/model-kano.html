KANO

KANO Model
一把需求筛子，把用户需求分五类——不是所有「用户要的」都值得做，分错类就白忙。
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： KANO 模型像一把需求筛子，把用户需求分了五类——基本、期望、兴奋、无差异、反向。它告诉你力气该往哪使：不是所有「用户要的」都值得做，分错类就白忙。
Origin
狩野纪昭东京理工大学教授，提出 KANO 需求分类模型。
Field
产品 / 需求管理凡是「分不清该做哪个需求」的团队都用得上。
Core Definition
需求五分类基本 · 期望 · 兴奋 · 无差异 · 反向，按满意度敏感度分层。
Thinking Layer
商业型把有限资源压在最能撬动满意度的那层需求上。

误区 vs 模型视角

Bad vs Good
✕

常见误区

Satisfying the Wrong Layer
  • 加了功能没人夸、少了被骂，还硬做
  • 把基本需求当卖点拿去宣传
  • 基本需求没垫好，就急着上兴奋需求
  • 不痛不痒的小优化，加班做了半年
例：基础同步都经常丢，却花大力气做花哨社区广场——老用户一边骂卡顿一边走。
✓

模型视角

Floor, Then Scale, Then Gift
  • 先地板（基本）再天平（期望）最后拆礼包（兴奋）
  • 基本需求稳了，再悄悄加兴奋当惊喜
  • 反向和无差异需求，做了反扣分或纯白做
  • 定期重测，期望会变基本、兴奋会变无差异
例：同步稳、搜索快（期望做满），再悄悄加「语音速记」当惊喜，口碑一下起来。

五类需求拆解

The Five Types
01

基本需求 · 地板

没有就死，有了也不加分。像酒店干净床铺，脏了被骂，干净了没人夸——是及格线，不是卖点。

02

期望需求 · 天平

做得好就加分，做得差就掉粉。手机续航、App 速度都在这一层，用户横向比价，你弱一点他就走。

03

兴奋需求 · 惊喜礼包

没有不怪你，有了狂爱你。当年 iPhone 多点触控就是这种，用户没想过要，给了就上头，是暗处胜负手。

04

反向需求

做了反扣分。强行加广告弹窗就是典型——用户原本挺好，你一操作反而惹人烦。

05

无差异需求

做了纯白做，用户根本不在意。不痛不痒的小优化往往落在这层，忍住别动手。

三步落地

Put It to Work

列出所有需求

挨个放进五类格子（基本 / 期望 / 兴奋 / 无差异 / 反向），先看清家底。

先保基本、补期望

这两层没稳，别碰兴奋。地板没铺好，礼包全是空中楼阁。

定期重测

期望会变基本、兴奋会变无差异（Wi-Fi 免费从前是惊喜，现在是地板）。

模型金句

In a Nutshell

需求不是越多越好，是把对的那层做到极致，其余的忍住别动。

岛主批注

基本需求是地板，没有就死，有了也不加分——别拿来当宣传卖点。

地板的真相

先做满期望，再悄悄加兴奋，口碑才起得来；反过来，全是空中楼阁。
