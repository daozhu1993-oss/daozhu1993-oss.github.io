Thinking Model · No.01

MECE 原则

Mutually Exclusive, Collectively Exhaustive
巴巴拉·明托  ·  Barbara Minto  ·  《金字塔原理》· 麦肯锡方法论
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： MECE 是「相互独立、完全穷尽」的分类纪律——把问题拆成不重叠、不遗漏的几块，是麦肯锡式结构化思考的底层地基。它不替你想出答案，但能确保你既没在重复劳动，也没漏掉盲区。
Origin
巴巴拉·明托Barbara Minto，麦肯锡首位女性顾问，1970 年代在《金字塔原理》中系统提出。
Field
结构化思维 / 商业分析咨询、战略、产品与日常管理中的「拆解」底层方法论。
Core Definition
两大分类铁律相互独立（Mutually Exclusive）+ 完全穷尽（Collectively Exhaustive）。
Thinking Layer
拆解层 / 结构化在「系统 2 慢思考」之上，再叠加一道分类纪律。

误用 vs 善用

Bad vs Good
✕

非 MECE 思维

Overlapping & Leaky
  • 分类彼此重叠，同一元素脚踏两条船
  • 总有遗漏，关键部分藏在「其他」里
  • 维度混搭，想到哪切到哪
  • 分析重复、决策出现盲区
例：把用户分成「新用户 / 老用户 / 高价值用户」——高价值用户可能既是新也是老，三类相互重叠。
✓

MECE 思维

Mutually Exclusive & Exhaustive
  • 维度单一清晰，块与块互不重叠
  • 所有分类相加 = 全集，没有漏网
  • 用稳定维度切：时间 / 结构 / 流程 / 公式
  • 可逐层下钻，每层都过检验
例：按「注册时长 × 消费额」两维度画矩阵，每个用户恰好落在一个格子，且覆盖全部。

四个关键要点

The Four Disciplines
01

相互独立（ME）

任何一条信息只属于一个分类，不脚踏两条船。重叠是重复劳动和口径打架的根源。

02

完全穷尽（CE）

所有分类加起来必须等于全集。出现大块「其他」前，先确认它真的小到可以忽略。

03

先定维度，再拆分

用稳定维度切（时间 / 结构 / 流程 / 公式 / 人群），一次只用一个主维度，别混搭。

例：利润 = 收入 − 成本，沿变量拆，天然 MECE。
04

画树状图 + 检查

把层级画成树，每一层都问一遍：有重叠吗？漏了吗？可视化让盲区无所遁形。

应用流程

From Problem to MECE

明确问题边界

先搞清楚「到底在拆什么」，别上来就分。边界模糊，分类必乱。

选一个主维度

时间 / 结构 / 流程 / 公式 / 人群，挑一个最贴合的，一次只用一个。

第一层拆解

把全集切成 3–5 个互斥块，太少漏、太多乱，黄金区间在 3–5。

检查 MECE

逐块问：重叠？遗漏？补「其他」前先确认它是真的小头。

逐层下钻

对关键块重复 2–4 步，直到每一块都能直接行动为止。

模型金句

In a Nutshell

如果你没法把一个议题拆成互不重叠、又完全穷尽的部分，说明你还没真懂它。

MECE 的判别标准

好的分类不是为了好看，是为了让该被看见的盲区，无处可藏。

分类的真正目的

结构清晰，不是思考的装饰，而是思考的证据。

岛主批注

马上能用的事

Put It to Work
✓

汇报前 MECE 自查

我的分类有重叠吗？有遗漏吗？两条都过，再进会议室。

✓

警惕「其他」

看到大块"其他"先停一下——它往往是被你随手丢掉的盲区。

✓

公式法找维度

利润=收入−成本、增长=新客+回流，沿变量拆，天然互不重叠。

✓

用户分群先定单一维度

先用生命周期切，别把"年龄/价值/渠道"混在一起，否则必然重叠。

✓

复杂问题先画一棵树

每画一层就问"这一层 MECE 吗"，比在脑子里空想可靠得多。

✓

团队对齐同一张树

用同一张 MECE 树讨论，避免各切各的、口径永远对不上。

什么时候该掏出它

When to Use
工作汇报 项目拆解 用户分群 竞品分析 根因分析（RCA） 战略规划 面试结构化答题 任何"先分类再下手"的场景
