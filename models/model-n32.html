Thinking Model · No.139

边际效用

Marginal Utility
你每多消费一份同样的东西，它给你的满足感就递减一截，最后一口最不甜。
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 边际效用递减铁律说：同样的东西，你拥有的越多、消费的越频，新增的那一口快乐就越小。它不只是吃西瓜的常识，更解释了为什么水便宜钻石贵、为什么老夫老妻会平淡、为什么加薪第三次就无感。
Origin
奥地利学派（19世纪）素材指出由门格尔等提出；岛主解读 · 思维模型合集
Field
经济学 · 消费心理定价、激励、情感经营都用得上
Core Definition
每多消费一单位相同商品，新增满足感递减价值不取决于"多有用"，而取决于"当下还缺不缺"
Thinking Layer
认知 / 决策理解"再多也填不满"的满足规律

误用 vs 善用

Bad vs Good
✕

把"拥有"当"永远甜"

The More-Is-Better Trap
  • 以为加薪、加量、加爱就能一直让人开心，直到边际效用归零还硬加。
  • 把亲人的好当空气，吃到第六个包子还嫌不够香，忘了前面几个多珍贵。
  • 用同一招激励团队三次，第三次大家开始比"凭啥他加我没加"。
  • 在一段关系里无限索取新鲜感，却不懂"重置"感恩。
例：给团队连加三次薪，第一次干劲翻倍，第三次基本无感，反而有人因"不公平"而离职。
✓

把满足用在刀刃上

Spend Utility Wisely
  • 一点水先喝、多了洗菜、再多了洗澡——用途越往后越不重要，效用自然递减。
  • 激励换花样：首次加薪、二次升职、三次给战略旁听权，别老用同一招。
  • 感恩要"重置"，定期提醒自己跳出"第几个包子"的麻木。
  • 控制欲望上限，知道再多也填不满，反而能停下来享受当下。
例：沙漠快渴死时你选水，走出沙漠你选钻石——价值随"当下还缺不缺"而变。

三层拆解，看见递减

Three Layers
01

欲望有限，刺激会疲劳

听同一首歌第一遍起鸡皮疙瘩，听一百遍只想静音。消费越多，新增的爽感越微弱。

02

东西先用在最值的地方

稀缺资源按效用排序使用，越往后用途越不重要，整体效用自然越来越低。

03

它解释了"水便宜钻石贵"

一样东西的价值不取决于它多有用，而取决于你"当下"还缺不缺它。

04

激励也会边际递减

岛主亲测：首次加薪大家干劲翻倍，加第三次基本无感，甚至开始比较公平。

05

关系里的"第五口西瓜"

恋爱开头轰轰烈烈，几年后平淡如白水——不是不爱了，是边际效用在说话。

识别信号 → 应对

Signal → Response

识别"吃到没感觉"

当你对某件"本来很香"的事开始麻木、索取更多却更空，就是边际效用在报警。

做效用排序

把稀缺资源（时间、钱、注意力）先用在边际效用最高的事上，别平均撒。

让感恩"重置"

别把亲人的好当空气，定期提醒自己这是"第几个包子"之外的珍贵。

激励换花样

管理团队时轮换激励方式，别让同一招的边际效用跌到零。

设欲望上限

知道再多也填不满，主动在某个点停下来，反而留得住当下的甜。

马上能用的事

Put It to Work
✓

给感恩设"重置键"

每周挑一件被你当空气的好，认真说一次谢谢。

✓

把稀缺资源排个序

今天列三件最值得投入时间的事，先做边际效用最高的。

✓

给团队换种激励

如果连加薪都没用，试试授权、露脸、战略旁听权。

✓

设"享受上限"

再喜欢的事，吃到第几口就停，留个念想比撑到腻好。

✓

关系里制造"小别离"

定期给亲密关系一点距离，边际效用会自己回血。
