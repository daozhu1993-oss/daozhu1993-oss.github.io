六顶思考帽

Six Thinking Hats
一次只戴一顶帽子，把混乱的讨论拆成六道清晰的工序。
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 思考最大的敌人是混乱——逻辑、情绪、信息搅成一团，谁也说服不了谁。六顶思考帽把思考拆成六种角色，一次只戴一顶，像彩色打印机先分色再合成，会议效率直接翻倍。
Origin
爱德华·德·博诺（Edward de Bono）六顶思考帽提出者，平行思维先驱
Field
团队协作 · 决策沟通会议效率、平行思维训练
Core Definition
把思考拆成六种角色（帽色），一次只戴一顶让混乱的讨论变清晰、可收敛
Thinking Layer
决策 / 沟通协作中的思维分工

误用 vs 善用

Bad vs Good
✕

帽子戴串了

Mixed-Up Hats
  • 红帽硬要论证，把直觉变成逻辑争吵。
  • 一开场就戴黑帽，点子刚冒头就被拍死。
  • 六顶帽子同时戴，情绪、事实、批判搅成一团。
  • 没人戴蓝帽，会议没有主持、没有收口。
例：产品会刚说个想法，立马有人"这不行、成本太高"，全场陷入互怼，两小时没结论。
✓

一次一顶，分色合成

One Hat at a Time
  • 蓝帽先定规则，明确此刻用哪顶帽。
  • 红帽只凭直觉、限时30秒，不展开。
  • 绿帽尽情发散，黑帽放到最后才挑刺。
  • 黄帽先于黑帽，先看见价值再找风险。
例：挪威石油公司耗资10万美元/天的装配难题，用六顶帽12分钟解决；J.P.摩根砍掉80%会议时间。

六顶帽子怎么分工

Six Roles
01

白帽 · 摆事实

只放数据和客观信息，不夹带判断。先把"发生了什么"摆清楚，再谈别的。

02

红帽 · 说感受

30秒内凭直觉答，不准论证——一解释就变逻辑，帽子就戴串了。

03

黄帽 · 找价值

主动找机会和亮点，先看见"好在哪"，别一上来就否定一切。

04

黑帽 · 挑风险

必须放在黄帽之后，专挑漏洞和隐患，但别让它抢跑、别抢跑。

05

绿帽 · 搞创意

发散时间禁止说"不行"，让点子先长出来，再谈筛选。

06

蓝帽 · 控全场

主持人专属，定议程、排帽子顺序、到点收口，别让讨论散了。

岛主带队的六步节奏

Six-Step Cadence

蓝帽定规则

开场先宣布"现在用哪顶帽"，所有人统一频道，不跑题。

白帽摆事实

把已知数据、现状摊开，只陈述、不评判，先对齐信息。

绿帽狂想

只许发散，禁止任何"不行"，点子越多越好，先不刹车。

黄帽找亮点

从狂想里挑出有价值、能落地的方向，给创意一点光。

黑帽挑风险

最后才批判性审查，补漏洞、防翻车，别让否定过早杀创意。

蓝帽收口

主持人汇总结论、定下一步，两小时顶原来一下午。

马上能用的事

Put It to Work
✓

开会先定帽序

会前写下"蓝→白→绿→黄→黑→蓝"，照单走，不临时起意。

✓

红帽计时30秒

谈感受时计时，到点必须停，不准展开论证、不准变逻辑。

✓

创意环节禁"不行"

绿帽时段贴张纸条"禁止说不行"，先让点子飞一会儿。

✓

黑帽往后放

把批判环节挪到最后，别让否定过早杀掉好创意。

✓

收尾用蓝帽

指定一人做主持，负责总结结论和待办，别散会就散了。
