SECI 知识转化螺旋

SECI Knowledge Spiral
知识不是存进硬盘就叫拥有，是在人身上转一圈才活。
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 老师傅的手艺别人学不会，因为那是"说不清"的隐性知识。野中郁次郎用 SECI 模型讲明：组织靠"社会化→外显化→融合化→内隐化"四步螺旋，把心得变成本事，像酿酒一圈圈越酿越醇。
Origin
野中郁次郎（Ikujiro Nonaka），1995知识管理之父，SECI 模型提出者
Field
知识管理 · 组织学习隐性 / 显性知识转化
Core Definition
组织通过"社→外→融→内"四步螺旋，把隐性心得变显性本事让"我懂但说不出"变成"能传能练"
Thinking Layer
成长 / 组织团队能力沉淀

误用 vs 善用

Bad vs Good
✕

只发手册不给练

Manuals Without Practice
  • 以为把资料存进知识库，团队就拥有了能力。
  • 只做培训讲座，听完回去还是老样子。
  • 老手经验全在脑子里，从不外显成文档。
  • 新人看了 100 遍教程，却从不下水实操。
例：公司攒了一堆操作手册，新人背得滚瓜烂熟，一上手还是错——知识停在纸面，没转成自己的本能。
✓

四步螺旋转起来

The Four-Step Spiral
  • 社会化：让新手看师傅、跟师傅，在模仿里学长不出的手感。
  • 外显化：逼团队把心得讲成话、写成流程，变成能传的文档。
  • 融合化：把各处知识整合成体系，沉淀成组织智慧。
  • 内隐化：员工练熟了，又变回自己的本能反应。
例：新人跟老销售跑一周（社会化）→逼写签单话术（外显化）→月底汇成话术库（融合化）→新人照练出单（内隐化），一轮跳过半年坑。

螺旋的四道工序

Four Modes
01

社会化 S

隐性→隐性：观察、模仿、试错，学长不出的"手感"，像看人骑车看不会，得自己摔两跤。

02

外显化 E

隐性→显性：把心得讲成话、写成流程，变成能传的文档，知识才算离开脑子。

03

融合化 C

显性→显性：整合各处知识，做加减成体系，沉淀成"组织智慧"而非个人碎片。

04

内隐化 I

显性→隐性：员工练熟了，又变回自己的本能，像游泳看百遍不下水还是不会。

05

四个"场"撑着

创始场、交流场、虚拟场、实践场，断了螺旋就转不上，卡在某一圈上不去。

06

螺旋是循环的

一轮转完，新隐性又起新一轮，越酿越醇，不是一次就结束的单程。

带新人这样转一圈

Run One Loop

老手带新手

先观察模仿（社会化），跟岗不急着上手，把说不清的手感学长。

逼写经验

让团队把心得写成案例和流程（外显化），别只留在老手脑子里。

定期复盘

把各处知识整合成库（融合化），做组织的加法和减法。

新人实战

照着练、在做中学（内隐化），练出成果才算真正内化。

补"场"

看螺旋卡在哪一步，把对应的场补上，让圈连得上、转得动。

In a Nutshell

马上能用的事

Put It to Work
✓

给经验建档

本周逼团队把一条核心经验写成可传文档，别只存在某人脑子里。

✓

师徒跟岗

让新人先跟老手跑一轮，别只发 PDF，手感是看不会的。

✓

月度复盘库

每月把散落经验汇成团队知识库，做组织的加法。

✓

做中学考核

新人必须上手练出成果，才算真正内化，不下水永远不会。

✓

查"场"断点

看螺旋卡在哪一步，把对应的场补上，让圈转得动。
