GTD 时间管理

Getting Things Done (GTD)
大脑是用来思考的，不是用来记事的——给它外接一个「第二脑」。
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： 戴维·艾伦在《尽管去做》里给了解药：GTD 就是给大脑外接一个「第二脑」，把杂念清空，专注当下。收集→处理→管理→检查+行动，五步跑通。
Origin
戴维·艾伦（David Allen）2001 年《尽管去做》（Getting Things Done）提出 GTD 工作法
Field
效率 · 自我管理 · 工作流邮件洪流、多线程项目、脑子嗡嗡响时最管用
Core Definition
把脑子里所有待办倒进外部系统，按「下一步动作」分流处理，让大脑只负责思考、不参与记忆。收集→处理→管理→检查→行动 五步闭环
Thinking Layer
成长 · 执行从焦虑到心流

误用 vs 善用

Bad vs Good
✕

靠脑子当硬盘的「后台嗡嗡」

Mental Overload
  • 邮件会议待办堆成山，脑子总在后台嗡嗡响
  • 越想「别漏事」，越在关键时刻掉链子
  • 打开邮件又塞回「待会儿再说」，永远在篮子里
  • 靠记忆硬扛，洗澡时才蹦出「忘了回谁」
例：脑子里同时挂着 30 件事，结果开会时突然想起没回老板消息，当场心慌。
✓

外接「第二脑」的清空法

Getting Things Done
  • 把所有事倒进收集篮，别让它们占内存
  • 处理时一次一件、永不放回篮子
  • 四张清单分流：愿望/等待/项目/执行
  • 按环境、时间、精力、重要性挑当下最该做的
例：岛主用收集篮代替脑子记，反而能全心写东西进心流；微软、甲骨文都把它当成功习惯推。

关键要点

Key Points
01

收集：把脑子里所有事倒出来

拿张纸 20 分钟，鸡毛蒜皮到重大项目全写下来，别让它们占内存。

02

处理：一次一件，永不放回篮子

打开邮件就判定：立刻做（2 分钟内）、委派别人、还是进系统。最忌「待会儿再说」又塞回去。

例：2 分钟能做的，马上做掉，别让它回流。
03

管理：四张清单分流

愿望清单（以后再说）、等待清单（别人做）、项目清单（多步骤）、执行清单（自己干）。

04

检查 + 行动：按情境挑当下

按环境、时间、精力、重要性，挑当下最该做的一件，而不是被脑子随机提醒。

05

周五回顾：清空大脑过周末

周五花 1 小时回顾，把所有悬空的事归位，安心放下工作。

例：岛主亲测：用了收集篮，反而能全心写东西进心流。

应用流程

How to Apply

收集

花 20 分钟把所有待办扫进一个「工作篮」，从脑子里清空出来。

处理

对每件事问「下一步动作是什么」，2 分钟能做的马上做，其余判定去向。

分流

进四张清单之一：愿望 / 等待 / 项目 / 执行，各就各位不再乱飘。

检查

每周五花 1 小时回顾，确认每件事都有去处，清空大脑。

行动

按当下环境、时间、精力、重要性，挑清单里最该做的那一件，全心去做。

Put It to Work
✓

今天花 20 分钟扫进工作篮

把所有待办倒进一个收集工具，别让任何事还挂在脑子里。

✓

对每件事问「下一步动作」

处理时逼自己写出具体下一步，含糊的「跟进一下」不算数。

✓

建四张清单并分类

愿望 / 等待 / 项目 / 执行，把工作篮里的东西逐项归位。

✓

2 分钟能做的马上做

遇到两分钟内能解决的小事，立刻做掉，别让它回流篮子。

✓

每周五花 1 小时回顾

固定时间清空大脑，安心过周末，周一不慌。

✓

把脑子里的「待会儿」全写下来

下次冒出「别忘了」，立刻落到纸上，把内存腾给思考。
