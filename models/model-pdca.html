PDCA 循环模型

Plan · Do · Check · Act
岛主解读  ·  持续改进操作系统  ·  把任何事变成「越做越好」的引擎
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： PDCA 是一个永远转动的升级轮子——计划、执行、检查、处理四环咬合。它把一次性、线性的努力，变成循环迭代的精进，让每一次循环的终点，成为下一次更高水平循环起点。
Origin
岛主思维模型解读源自质量管理（戴明环 Deming Cycle），广泛用于企业与个人持续改进。
Field
自我管理 / 项目管理 / 执行力习惯养成、创作优化、学习计划都适用。
Core Definition
四环咬合 P·D·C·APlan 计划 / Do 执行 / Check 检查 / Act 处理。
Thinking Layer
框架型 / 迭代循环在「做一次」之上，叠加「转一圈」的纪律。

线性努力 vs PDCA 循环

Bad vs Good
✕

一次性、线性的努力

One-and-Done
  • 做完像交差，下次从头摸索
  • 定了计划，结果不如意，然后没然后了
  • 总在重复同样的错误
  • 只有 P 和 D，没有 C 和 A
例：定了健身计划，中断了就中断了，明年再「重新开始」。
✓

PDCA 循环迭代

Plan · Do · Check · Act
  • P 先沙盘推演，挖根因再定方案
  • D 执行中记录原始数据
  • C 把结果与目标对比，找偏差原因
  • A 标准化成功经验，遗留问题进下一轮
例：发现「晨间听」效果好，固定成标准动作；熬夜问题启动新循环去解决。

PDCA 四环拆解

The Four Phases
P

Plan 计划

不是空想，是沙盘推演。三问：改善什么（目标要清晰）？为什么不理想（深挖根因，别只看表面）？具体怎么做（步骤、资源、时间）？例：英语听力差 → 听得少方法错 → 每天 30 分钟精听 VOA。

D

Do 执行

把计划付诸实践，关键是有意识地记录过程和原始数据。「哪听不懂」「花多久」「心情如何」都记下来，Check 环节才有东西比。

C

Check 检查

复盘而非庆祝或懊悔。对比计划与目标：做得好吗？哪不一样？为什么偏差？例：一周后看完成几天、提升了没、为何两天没坚持。

A

Act 处理

最关键的升级环节。标准化成功经验（固化成标准流程）；把遗留问题纳入下一轮 P。例：把「早上听」固定，材料太难启动新循环。

极简启动法（以「每周运动 3 次」为例）

Make It Turn

P 花 5 分钟计划

为什么总中断？（下班晚、累）→ 改周二四六早 7 点在家练 20 分钟。

D 执行一周

闹钟一响就起，练！过程中随手记「今天状态如何、卡在哪」。

C 周末检查

成功了吗？周四因熬夜没起，为什么？睡前刷手机了。

A 立刻处理

固定「周二六顺利」的经验；为「熬夜」启动新循环——每晚 11 点手机放客厅。

模型金句

In a Nutshell

PDCA 的灵魂不在于计划多完美，而在于 C-A 环节——持续检查并勇敢调整。

关键在循环

每一次循环的终点，都是下一次更高水平循环的开始。

永动引擎

做完一件事就交差，是线性努力；转起这个圈，才是迭代精进。

岛主批注

马上能用的事

Put It to Work
✓

从一件小事转圈

别想太复杂，先拿「每周运动 3 次」跑通一个完整 PDCA。

✓

计划先问根因

P 阶段别只写「要更好」，先挖「为什么现状不理想」。

✓

执行时记数据

Do 阶段随手记原始数据，Check 才有东西比。

✓

周末做一次 Check

把结果和计划对比，找偏差和原因，别只庆祝或懊悔。

✓

把成功经验固定

A 阶段把「有效的」写成标准流程，下次不重头摸索。

✓

遗留问题马上开新圈

没解决的，直接丢进下一轮 P，循环不停。
