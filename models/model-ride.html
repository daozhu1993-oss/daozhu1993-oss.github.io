Thinking Model · No.02

RIDE 说服模型

Risk · Interest · Difference · Effect
岛主解读  ·  沟通说服力框架  ·  根植于「损失厌恶」行为心理学
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： RIDE 是「先避害、后趋利」的说服框架——利用人对损失的敏感远大于对获得的渴望，先讲风险打开对方的耳朵，再给利益、亮差异、坦影响，顺着人性的决策曲线，把人一步步说服。
Origin
岛主思维模型解读底层逻辑来自行为心理学中的「损失厌恶」（Loss Aversion）。
Field
沟通 / 说服 / 谈判 / 销售凡是要"让别人说 yes"的场合都用得上。
Core Definition
四步框架 R·I·D·ERisk 风险 / Interest 利益 / Difference 差异 / Effect 影响。
Thinking Layer
表达与影响层在"把事想清楚"之后，解决"怎么让人听进去"。

错误说服 vs RIDE 说服

Bad vs Good
✕

错误说服

Benefits-First, Deaf Ears
  • 一上来就狂讲好处，对方自动屏蔽
  • 忽略人性底层的"损失厌恶"
  • 只给甜头、没有痛感，对方无所谓
  • 越努力讲，对方越无动于衷
例：跟老板提加薪，罗列自己多努力、多值钱，老板一句"再考虑考虑"就打发了。
✓

RIDE 说服

Risk First, Then Reward
  • 先讲风险，激活"避害"心理，对方才认真听
  • 再给利益当"解药"，趋利＋避害双驱动
  • 点出差异，帮对方做"选你"的决策
  • 坦言影响，建立信任，消除最后一丝犹豫
例：先说"不调整会错过 618、损失 30% 潜在客户"，再给方案提转化，老板立刻坐直了。

RIDE 四步拆解

The Four Moves
R

风险 Risk

开门见山指出"不按你说的做"会面临什么损失。人对损失的恐惧远大于对获得的渴望，先激活避害心理，对方才打开耳朵。

例句：老板，这个月再不调整营销策略，可能会错过 618 流量红利，损失至少 30% 的潜在客户。
I

利益 Interest

陈述采纳你的建议后会获得什么具体好处。在对方已经"紧张"之后给解药，趋利＋避害叠加，效果翻倍。

例句：如果采纳我的方案，不仅能保住现有流量，还能靠精准投放把转化率提升 15% 以上。
D

差异 Difference

说明你的方案和其他选项比，独特在哪、为什么选你不选别人。人怕选错，差异点就是"选你"的安心理由。

例句：和传统广撒网不同，我这个方案聚焦私域，成本更低、用户粘性更高，特别适合我们现在的预算。
E

影响 Effect

主动说出方案可能存在的小缺点、小影响。人天生怀疑"完美的东西"，暴露小瑕疵反而增加信任，同时强调它无关紧要。

例句：当然，前期需要多投一点人力做内容，但一旦跑通，后期几乎零成本维护。

应用模板（直接填空）

The RIDE Template
R 风险：如果不 ______，我们会面临 ______ 的风险 / 损失。
I 利益：如果采纳我的建议，我们可以获得 ______ 的好处。
D 差异：和其他方案相比，我的建议独特在 ______。
E 影响：当然，它也有一个小缺点 ______，但这不影响大局，因为 ______。

模型金句

In a Nutshell

说服不是"把我想说的说完"，而是"把对方想听的按顺序给到"。

RIDE 的本质

先讲风险，再给解药——顺序错了，再对的道理也进不了耳朵。

为什么 R 必须排第一

主动暴露一个小缺点，比假装完美更能让人信你。

Effect 的玄机

马上能用的事

Put It to Work
✓

提加薪前先写"不做的损失"

把"我会更努力"换成"不调整会损失什么"，老板的耳朵立刻打开。

✓

"好处清单"挪到风险之后

任何时候说服，先痛后甜，中间再亮差异，顺序别乱。

✓

备好一句"为什么选我不选别人"

D 步骤最容易被省略，却是帮对方下决心的关键。

✓

主动说一个无伤大雅的缺点

别等对方发现，抢先坦白小瑕疵，信任感反而拉满。

✓

用模板填空，别临场发挥

下次要说服人，先把 RIDE 四空填好，照着说就稳。

✓

说服前自检：顺序是 R→I→D→E 吗

只要顺序对，内容差一点也管用；顺序错，内容再好也白搭。

When to Use
