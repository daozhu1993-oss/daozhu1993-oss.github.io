RPV 框架

RPV Framework
克里斯坦森  ·  Clayton Christensen  ·  《创新者的解答》
一句话总览： 一家公司的资源、流程、价值观，决定了它能做什么、不能做什么。资源越多越舍不得放弃旧赛道，流程偏爱"和过去一样"，而真正卡死创新的，往往是那条"什么才值得做"的价值观。
Origin
克里斯坦森Clayton Christensen，在《创新者的解答》中提出 RPV 框架。
Field
创新管理 / 组织能力凡要回答"为什么大公司做不出新东西"都用得上。
Core Definition
R·P·V 三件套资源 Resources / 流程 Processes / 价值观 Values。
Thinking Layer
能力层 / 组织约束在"想创新"之下，看清是什么在暗中焊死你。
资源多=能创新？ vs RPV 视角 Bad vs Good
✕

以为有钱就能创新

Resource = Innovation?
  • 钱、人、技术都不缺，却动不了
  • 用主业 KPI 和价值观考核新业务
  • "不熟不做"把机会挡在门外
  • 把公司活成一座博物馆
例：一家硬件公司账上躺一个亿，每次做软件，流程就要求"先看毛利率"，三年新业务一毛没长。
✓

RPV 视角

Resources · Processes · Values
  • 看清 RPV 三件套如何联手焊死你
  • 给创新开"特区"，别用主业尺子量
  • 把"该不该做"的判据写下来
  • 定期问：资源在成就你，还是定义你
例：独立小队、独立考核、独立价值观，新业务才挣脱 RPV 的绳子。

三件套拆解

R · P · V
R

资源 Resources

人、设备、技术、品牌、现金、客户关系……拥有的越多，越舍不得放弃旧赛道，像穿了一身名牌舍不得下泥地。

陷阱：资源是盔甲，也是包袱。
P

流程 Processes

为出好产品定下的流程和 KPI。它是效率的发动机，也是创新的紧箍咒——天然偏爱"和过去一样"的事。

陷阱：流程让老事更顺，让新事更难。
V

价值观 Values

组织默认的决策标准。真正卡死创新的，往往不是能力，而是"什么才值得做"的判断（财务标准引导型最典型）。

陷阱："不熟不做"四个字，挡掉无数机会。

应用流程

Unhook the RPV

给创新开特区

别用主业 KPI 和价值观考核它，让新业务有自己的尺子。

写清"该不该做"

把判据白纸黑字写下来，看清自己被哪条价值观绑架。

定期自我追问

问：我们在用资源成就新事，还是被资源定义成旧人？

看三件是否联手

RPV 是不是又悄悄焊死你，早发现早松绑。

模型金句

In a Nutshell

资源让你站上今天，流程和价值观却可能让你错过明天。

RPV 框架的核心判断

它不缺能力，是 RPV 三件套联手把它焊死在原地。
