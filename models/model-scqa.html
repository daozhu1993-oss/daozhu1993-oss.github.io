SCQA 模型

Situation · Complication · Question · Answer
岛主解读  ·  结构化表达框架  ·  麦肯锡都在用的「故事线」
← 返回思维模型图鉴 ← 回岛主主页
一句话总览： SCQA 是一条最符合大脑思考习惯的表达故事线——先铺情境建立共识，再抛冲突打破平静，自然引出问题，最后给出答案。它不是内容的替身，而是让好内容「听得进去」的骨架。
Origin
岛主思维模型解读源自麦肯锡金字塔原理的结构化表达框架（Situation-Complication-Question-Answer）。
Field
沟通表达 / 工作汇报 / 文案写作演讲、同步信息、争取资源都用得上。
Core Definition
四段故事线 S·C·Q·A情境 → 冲突 → 问题 → 答案。
Thinking Layer
框架型 / 叙事结构在「想清楚」之后，解决「怎么讲清楚」。

SCQA 结构表达

Situation · Complication · Question · Answer
  • S 先建共识：一个双方都认同的背景
  • C 再打破平静：点出现实与理想的落差
  • Q 自然引出：此刻大家心里的核心问题
  • A 顺理成章：有铺垫，答案备受期待
例：灰指甲广告——你有灰指甲(S) → 一个传染俩(C) → 问我怎么办(Q) → 用亮甲(A)。

SCQA 四段拆解

The Four Moves
S

Situation 情境

建立共识。描述你与听众都认同的背景，简洁、客观、无争议。例：「本季核心目标是提升用户活跃度 20%。」

C

Complication 冲突

打破平静。点出现实与理想的落差、变化或挑战，制造张力，回答「为什么现在要谈」。例：「新用户次日留存下降了 15%。」

Q

Question 问题

引出焦点。基于冲突，提出大家此刻都会产生的核心问题，精准指向你要给的答案。例：「如何有效提升留存？」

A

Answer 答案

给方案。你整个表达的核心观点，因前面铺垫显得顺理成章、备受期待。例：「启动新用户护航计划，从三方面着手……」

应用流程

Build Your SCQA

定 S 情境

找一个双方都认同、无争议的事实背景，越简洁越好，这是共同出发点。

造 C 冲突

点出与 S 的落差或变化，制造「为什么现在要谈」的张力。

抛 Q 问题

顺着冲突，抛出一个听众心里自然冒出的核心问题。

给 A 答案

用最强结论回应 Q，此时答案已被期待、顺理成章。

ASC 开门见山式：答案 → 情境 → 冲突（适合领导已知背景、时间紧）
CSA 突出忧虑式：冲突 → 情境 → 答案（适合争取资源、强调危机感）
QSCA 突出信心式：问题 → 情境 → 冲突 → 答案（适合演讲、文案开头引好奇）

模型金句

In a Nutshell

清晰的表达，本质上是对听众认知过程的善意引导。

SCQA 的本质

从是什么（S），到怎么了（C），再到为什么（Q），最后怎么办（A）——跟着走，没人说你讲不清。

故事线节奏

模型是死的，人是活的；顺序可以调，但 S-C-Q-A 的底层逻辑别丢。

马上能用的事

Put It to Work
✓

开口前先写 S-C-Q-A

关键对话 / 汇报前，把四步在纸上列一遍再开口。

✓

S 要双方都认

情境别夹带争议，否则共识没建起，后面全塌。

✓

C 必须戳痛点

没有冲突就没有张力，听众不会关心你的答案。

✓

Q 要对准 A

你抛的问题，必须被后面答案精准接住。

✓

赶时间用 ASC

领导已知背景，直接给答案再补情境冲突，省时。

✓

文案开头试 QSCA

想抓人，先抛「如何用一年改变自己？」再展开。
